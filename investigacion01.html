Usabilidad Web

¿Qué es?
Surgió a partir del desarrollo y del nacimiento de internet como red de comunicación, formalmente la usabilidad se desarrolló a partir del importante trabajo de Jakob Nielsen este considerado el padre de la Usabilidad. Este término ha sido utilizado para aquellos factores que ayudan a que un sitio web sea fácil de utilizar. El principal protagonista de este término es el usuario como tal, lo que permite que se le denomine diseño centrado en el usuario.

El estándar ISO 9241-11 que forma parte de la serie ISO 9241, define la usabilidad como "la medida en la que un producto se puede usar por determinados usuarios para conseguir objetivos específicos con efectividad, eficiencia y satisfacción en un contexto de uso especificado".

Importancia

Es uno de los elementos cruciales a la hora de construir una nueva página web, es importante porque ayuda a mejorar la experiencia de usuario lo que garantiza que los usuarios encuentren exactamente lo que están buscando sin esfuerzo, aumenta la eficiencia de la página lo que permite que los usuarios aumenten su productividad y el ahorro del tiempo, genera confianza lo que inspira a los usuarios a inclinarse más con la marca y ser más activos, incrementa la retención de los usuarios con un diseño fácil e intuitivo que fomente la lealtad de los usuarios.

A pesar de los esfuerzos actuales, muchos sistemas aún presentan una usabilidad deficiente. Esto se debe a que se enfocan más en aspectos internos como el rendimiento y la confiabilidad, dejando en segundo plano elementos cruciales como el diseño interactivo centrado en el usuario, la adaptación a las características específicas de cada usuario y la rápida presentación de la información solicitada.

Principios

Propuestos por Jakob Nielsen:

Visibilidad del estado del sistema El usuario debe saber en todo momento lo que está sucediendo, ya sea en el sitio o la aplicación web, por ejemplo, barras de carga, íconos giratorios, a la hora de subir o descargar archivos un porcentaje con el progreso en tiempo real.
Relación entre el sistema y el mundo real Se debe emplear en el desarrollo del sistema un lenguaje que sea entendible para los usuarios, al ser desarrolladores tenemos que pensar como usuarios como por ejemplo la utilización del carrito de comprar asimilando un carrito de un supermercado. .
Control y libertad de usuario En la mayoría del tiempo se le debe dar al usuario la posibilidad tanto como de deshacer o rehacer sus acciones, el más claro ejemplo son los famosos íconos con las flechas hacia atrás y hacia delante, como los de las aplicaciones de Office.
Consistencia y estándares : Ya hay estándares impuestos para ciertas partes de lo que es una página web, solo hay que ser consistentes con ellos, por ejemplo, el ícono de las tres rayitas que significa el menú o también el carrito de compras.
Prevención de errores Es uno de los más importantes, se basa en poder adelantarse el error y antes de que pase poder prevenirlo, un ejemplo puede ser a la hora de enviar un correo, cuando ya tenemos todo el cuerpo completo, pero no hemos puesto el destinatario este no nos dejará enviar el correo hasta proporcionarlo.
Reconocimiento antes que recuerdo El usuario debe poder recordar fácilmente sus datos y sus acciones que realiza, como un ejemplo páginas famosas como Amazon que están divididas en secciones y con imágenes lo que permite la ubicación del usuario de manera sencilla.
Flexibilidad y eficiencia de uso Nuestro sitio debe ser eficiente y flexible tanto para los usuarios que están familiarizados con los procesos como para los que no, como ejemplo algunas herramientas de edición de imágenes poseen íconos representativos que son de fácil guía para cualquier tipo de usuario.
Estética y diseño minimalista Es bueno optar por esta opción porque así el sitio web será atractivo visualmente para los usuarios, y no provocará fatiga visual a largo plazo, como por ejemplo páginas de joyería o tiendas sofisticadas adoptan este tipo de diseño.
Ayudar al usuario a reconocer, diagnosticar y recuperarse de errores Si no se pudo prevenir el error hay que tener un plan de contingencia para saber que es el procedimiento a seguir, como por ejemplo el famoso error 404, este se puede modificar para que cuando ocurra no se le muestre el mensaje de error al usuario, sino que simplemente lo devuelva a la página de inicio.
Ayuda y documentación Es necesario tener siempre una parte de ayuda o preguntas frecuentes, para que cuando el usuario no sepa que hacer al menos tenga un lugar donde acudir, como ejemplo se puede crear un foro donde estén todas las preguntas y respuestas creadas por los usuarios y también la documentación necesaria para la usabilidad del sistema
